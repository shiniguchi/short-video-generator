**app/services/landing_page/optimizer.py** — Add `inject_analytics_beacon()` function at the end of the file:

```python
BEACON_TEMPLATE = """<script>
(function(){
  var w="%s";var id="%s";
  if(!w)return;
  var b=function(e,d){navigator.sendBeacon(w+"/track",JSON.stringify(d));};
  b("pageview",{lp_id:id,event:"pageview",referrer:document.referrer||"direct"});
  var f=document.querySelector("form");
  if(f)f.addEventListener("submit",function(){b("form_submit",{lp_id:id,event:"form_submit"});});
})();
</script>"""


def inject_analytics_beacon(html: str, worker_url: str, lp_id: str) -> str:
    """Inject analytics beacon script before </body>. Call AFTER optimize_html()."""
    if not worker_url:
        return html  # No tracking in local dev
    beacon = BEACON_TEMPLATE % (worker_url.rstrip("/"), lp_id)
    return html.replace("</body>", beacon + "</body>", 1)
```

Use `%s` formatting (not `.format()`) to avoid conflicts with JS curly braces.

The beacon:
- Fires pageview on load via `navigator.sendBeacon`
- Attaches form submit listener to fire form_submit event
- Sends referrer with pageview event (ANLYT-05)
- Skips tracking if worker_url is empty (guard: `if(!w)return;`)

**IMPORTANT**: Do NOT modify `generator.py` to call `inject_analytics_beacon()` during generation. Beacon injection happens at *deploy time* (Phase 19), not at *generation time*. This matches DEPLOY-04: "LP includes analytics beacon script before deployment." The function is available for Phase 19's deploy flow to call after reading the HTML.
