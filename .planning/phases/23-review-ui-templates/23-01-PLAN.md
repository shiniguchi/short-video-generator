Add two new routes at the bottom (before `_run_generation`):

1. `GET /ugc` route:
```python
@router.get("/ugc", response_class=HTMLResponse)
async def ugc_list(request: Request, session: AsyncSession = Depends(get_session)):
    """List all UGC jobs."""
    result = await session.execute(select(UGCJob).order_by(UGCJob.created_at.desc()))
    ugc_jobs = result.scalars().all()
    return templates.TemplateResponse(request=request, name="ugc_list.html", context={"ugc_jobs": ugc_jobs})
```

2. `GET /ugc/new` route:
```python
@router.get("/ugc/new", response_class=HTMLResponse)
async def ugc_new(request: Request):
    """Serve UGC job creation form."""
    return templates.TemplateResponse(request=request, name="ugc_new.html", context={})
```

Note: The form POSTs directly to `/ugc/jobs` (the API endpoint in `ugc_router.py`). That endpoint returns JSON `{"job_id": N, "status": "running"}`. After the POST, the user needs to be directed to the review page. Since the API returns JSON (not a redirect), add a small `<script>` block in `ugc_new.html` that intercepts the form submit, posts via `fetch()`, reads the `job_id` from the response, and redirects to `/ui/ugc/{job_id}/review`. This is simpler than adding a UI wrapper POST for job creation.

Script in `ugc_new.html`:
```html
<script>
document.querySelector("form").addEventListener("submit", function(e) {
  e.preventDefault();
  var fd = new FormData(this);
  fetch("/ugc/jobs", { method: "POST", body: fd })
    .then(function(r) { return r.json(); })
    .then(function(data) { window.location.href = "/ui/ugc/" + data.job_id + "/review"; })
    .catch(function(err) { alert("Error: " + err.message); });
});
</script>
```
