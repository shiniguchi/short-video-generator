# Section-scoped edit schemas (Phase 15: AI Section Editing)

class HeroEditCopy(BaseModel):
    """Copy fields for hero section editing."""
    headline: str       # 5-8 words, benefit-driven
    subheadline: str    # 15-25 words
    cta_text: str       # 2-4 words
    trust_text: Optional[str] = None

class BenefitsEditCopy(BaseModel):
    """Copy fields for benefits section editing."""
    benefits: List[dict]  # Each: {title, description, icon_emoji}

class FeaturesEditCopy(BaseModel):
    """Copy fields for features section editing."""
    features: List[dict]  # Each: {title, description, stat}

class HowItWorksEditCopy(BaseModel):
    """Copy fields for how_it_works section editing."""
    how_it_works: List[dict]  # Each: {step_number, title, description}

class CtaRepeatEditCopy(BaseModel):
    """Copy fields for cta_repeat section editing."""
    headline: str
    subtext: str
    cta_text: str
    urgency_text: Optional[str] = None

class FaqEditCopy(BaseModel):
    """Copy fields for FAQ section editing."""
    faq_items: List[dict]  # Each: {question, answer}

class WaitlistEditCopy(BaseModel):
    """Copy fields for waitlist section editing."""
    cta_text: str
    social_proof_text: str
    trust_text: Optional[str] = None

class FooterEditCopy(BaseModel):
    """Copy fields for footer section editing."""
    footer_text: str

**Step B: Create `app/services/landing_page/section_editor.py`**

This is the core Phase 15 module. ~120 lines. Must include:

1. **EDITABLE_SECTIONS** dict mapping section name -> (Pydantic schema, template context builder).
   - Covers: hero, benefits, features, how_it_works, cta_repeat, faq, waitlist, footer
   - Excludes: gallery (image paths, not copy)

2. **`get_editable_sections() -> List[str]`** — returns list of editable section names.

3. **`list_sections(html: str) -> List[str]`** — extracts all `data-section` values from HTML via regex. For `--list` discovery.

4. **`edit_section(html_path, section_name, user_prompt, product_idea, use_mock=False) -> dict`** — main entry point:
   - Read HTML from `html_path`
   - Validate: section exists in HTML (check `data-section="NAME"`)
   - Validate: section is editable (not gallery)
   - Extract current section context via `_extract_section_context()`
   - Generate new copy via `_generate_section_copy()` using `get_llm_provider().generate_structured()` with the section-scoped schema
   - Build template context from the generated copy via `_build_template_context()`
   - Re-render section via `render_section(section_name, template_context)`
   - Replace section in HTML via `_replace_section()` (regex on `data-section`)
   - Re-optimize via `optimize_html()`
   - Validate via `validate_html()`
   - Write back to `html_path`
   - Return `{"success": True, "html_path": str, "section": str, "warnings": list}`
   - On errors return `{"success": False, "error": str}`

5. **`_extract_section_context(html, section_name) -> str`** — regex extracts inner HTML of the target section.

6. **`_replace_section(html, section_name, new_html) -> str`** — regex replaces entire `<section data-section="NAME">...</section>` block. Uses `re.DOTALL`.

7. **`_generate_section_copy(section_name, user_prompt, product_idea, current_context, use_mock) -> BaseModel`**:
   - Look up schema from EDITABLE_SECTIONS
   - Build prompt: include product_idea, current section HTML context, and user's edit instruction
   - System prompt: "You are a landing page copywriter. Edit only the requested section. Preserve the product's voice and tone."
   - Call `get_llm_provider().generate_structured(prompt, schema, system_prompt, temperature=0.8)`
   - For `use_mock=True`: return a hardcoded instance of the schema with placeholder copy (similar pattern to `get_mock_copy()` in copy_generator.py)

8. **`_build_template_context(section_name, copy) -> dict`** — converts Pydantic model fields into the dict format that `render_section()` expects. Must match the `section_contexts` dict in `template_builder.py`:
   - hero: {headline, subheadline, cta_text, trust_text, video_url: None, hero_image: None}
   - benefits: {heading: "Why This Product?", benefits: [...]}
   - features: {features: [...], product_name: "this product"}
   - how_it_works: {steps: [...]}
   - cta_repeat: {headline, subtext, cta_text, urgency_text}
   - faq: {faq_items: [...]}
   - waitlist: {cta_text, social_proof_text, trust_text}
   - footer: {footer_text}

   NOTE: hero's video_url and hero_image are set to None because section editing only changes copy, not media. The template will render without the video/image — this is acceptable since the section replacement regex replaces the entire section block. If preserving media is needed, extract media URLs from current section context first. For MVP, copy-only editing is sufficient.
