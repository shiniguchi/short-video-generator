**1. Create app/services/landing_page/optimizer.py:**
- `def optimize_html(html: str) -> str`: Takes raw HTML string, extracts all `<style>` blocks, concatenates CSS, minifies with rcssmin.cssmin(), replaces multiple style blocks with single minified `<style>` in `<head>`. Returns optimized HTML.
- `def validate_html(html: str) -> dict`: Quick validation — checks for: viewport meta present, at least one h1, form element present, no external stylesheet links. Returns dict with {valid: bool, warnings: list[str]}.
- `def get_html_size_kb(html: str) -> float`: Returns size in KB. Warn if > 100KB.
- Use `import rcssmin` and `import re` for CSS extraction.

**2. Create app/services/landing_page/generator.py — the orchestrator:**
- `async def generate_landing_page(request: LandingPageRequest, use_mock: bool = False) -> LandingPageResult`:
  - This is the main end-to-end function. Orchestrates the full pipeline:
    1. **Research**: Call `research_lps(request.industry, request.region, use_mock=use_mock)` to get LP patterns
    2. **Color**: Call `get_color_scheme(preference=request.color_preference or "research", image_path=request.hero_image_path, research_patterns=research_result.patterns, preset_name=request.color_preset)`
    3. **Copy**: Call `generate_copy(request.product_idea, request.target_audience, research_result, use_mock=use_mock)` to get LP copy
    4. **Build**: Call `build_landing_page(copy, color_scheme, video_url=request.video_path, hero_image=request.hero_image_path)`
    5. **Optimize**: Call `optimize_html(raw_html)` to minify CSS and validate
    6. **Save**: Write to output path. Use `output/{run_id}/landing-page.html` where run_id is generated UUID hex[:8]. Create output dir if not exists.
    7. **Return**: LandingPageResult with html_path, product_idea, color_scheme, sections list
  - Log each step with timing information.
  - Use `from app.config import get_settings` for output_dir base path.

- `def generate_landing_page_sync(request: LandingPageRequest, use_mock: bool = False) -> LandingPageResult`:
  - Synchronous wrapper using `asyncio.run()` for CLI usage.

**3. Update app/services/landing_page/__init__.py:**
- Export: generate_landing_page, generate_landing_page_sync, LandingPageRequest (from schemas).
- Make it the clean public API.

**4. Create scripts/generate_landing_page.py — standalone CLI:**
- Follow the pattern from scripts/run_ugc_pipeline.py (sys.path setup, DATABASE_URL override, logging).
- Accept command-line arguments via argparse:
  - `--idea` (required): Product idea string
  - `--audience` (required): Target audience string
  - `--industry` (optional): Industry for LP research
  - `--region` (optional): Region for LP research (default: "US")
  - `--color` (optional): Color scheme preference — "extract"/"research"/"preset" (default: "research")
  - `--preset` (optional): Preset palette name (when --color=preset)
  - `--video` (optional): Path to video file for hero embed
  - `--image` (optional): Path to hero image for fallback
  - `--mock` (flag): Use mock data for all AI calls (default: uses USE_MOCK_DATA from env)
  - `--no-open` (flag): Skip opening browser after generation
- Build LandingPageRequest from args.
- Call generate_landing_page_sync(request, use_mock=args.mock).
- Print output path.
- Open in browser: `webbrowser.open(f"file://{os.path.abspath(result.html_path)}")` unless --no-open.
- Print file size and generation summary.

**5. Integrate into existing pipeline (app/pipeline.py):**
- Read the existing pipeline.py first. Add an optional LP generation step at the end.
- Add a function `generate_lp_from_pipeline(run_context: dict) -> Optional[str]`:
  - Extracts product_idea, target_audience from run context (these should already be available in the pipeline run data).
  - Finds video and hero image paths from the pipeline output.
  - Builds LandingPageRequest reusing pipeline assets.
  - Calls generate_landing_page_sync.
  - Returns the LP HTML file path.
  - This is called at the END of the pipeline, after video composition — per user decision "triggered as last step of existing video pipeline".
- Do NOT break existing pipeline flow. LP generation is additive, not replacing anything. If LP generation fails, log a warning and continue (don't fail the whole pipeline).
