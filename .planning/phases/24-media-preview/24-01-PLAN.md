  <name>Task 1: Register media_url filter and add media CSS</name>
  <files>app/ui/router.py, app/ui/static/ui.css</files>
  <action>
In `app/ui/router.py`, add a `_media_url` function after the `templates = Jinja2Templates(...)` line (around line 27):

```python
def _media_url(path: str) -> str:
    """Convert stored relative path 'output/foo/bar.mp4' to URL '/output/foo/bar.mp4'."""
    if not path:
        return ""
    return "/" + path.lstrip("/")

templates.env.filters["media_url"] = _media_url
```

This goes right after the `templates = ...` line, before the `_jobs` dict.

In `app/ui/static/ui.css`, add at the end:

```css
/* Media preview in review cards */
.media-card {
  padding: 12px;
}
.media-preview {
  width: 100%;
  max-width: 100%;
  border-radius: 4px;
  display: block;
  background: #000;
}
.media-preview-video {
  max-height: 320px;
  object-fit: contain;
}
```

Why `max-height: 320px`: generated clips are 9:16 (vertical). Without a height cap, each card becomes ~444px tall at 250px grid width, making the page extremely long.

Why `background: #000`: letterboxing for vertical video that doesn't fill card width.

Key rules:
- Always use `preload="metadata"` (not `auto`) — loads only duration + first frame
- Always include `type="video/mp4"` on `<source>` — avoids content-type probe round-trip
- Always check `{% if path %}` before `| media_url` — paths may be None
- Use `loading="lazy"` on `<img>` tags — prevents loading offscreen images
- Add `media-card` class alongside `stage-card` for padding
- Add both `media-preview` and `media-preview-video` classes on `<video>` elements
