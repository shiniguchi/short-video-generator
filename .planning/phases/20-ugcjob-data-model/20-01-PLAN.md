```python
from statemachine import StateMachine, State
```

States (match UGCJob.status values exactly):
- `pending` ‚Äî initial=True
- `running`
- `stage_analysis_review`
- `stage_script_review`
- `stage_aroll_review`
- `stage_broll_review`
- `stage_composition_review`
- `approved` ‚Äî final=True
- `failed` ‚Äî final=True

Transitions:
- `start`: pending -> running
- `complete_analysis`: running -> stage_analysis_review
- `approve_analysis`: stage_analysis_review -> running (back to running for next stage)
- `complete_script`: running -> stage_script_review
- `approve_script`: stage_script_review -> running
- `complete_aroll`: running -> stage_aroll_review
- `approve_aroll`: stage_aroll_review -> running
- `complete_broll`: running -> stage_broll_review
- `approve_broll`: stage_broll_review -> running
- `complete_composition`: running -> stage_composition_review
- `approve_final`: stage_composition_review -> approved
- `fail`: pending.to(failed) | running.to(failed) | stage_analysis_review.to(failed) | stage_script_review.to(failed) | stage_aroll_review.to(failed) | stage_broll_review.to(failed) | stage_composition_review.to(failed) ‚Äî any non-final state can fail

**Usage pattern (document in docstring):**
```python
sm = UGCJobStateMachine(model=job, state_field="status", start_value=job.status)
sm.send("start")  # validates pending -> running; writes job.status = "running"
```

**Important:**
- The `fail` transition must be reachable from ANY non-final state. Use the `|` (OR) syntax: `pending.to(failed) | running.to(failed) | ...`
- Add a docstring explaining: DB column is source of truth, instantiate with `start_value=job.status` for existing rows, state machine is a guard layer only
